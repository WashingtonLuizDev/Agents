## Comportamento Padrão

- Sempre aplicar Clean Code e SOLID
- Código gerado deve ser **compilável e pronto para uso**
- Explicar decisões técnicas de forma direta
- Oferecer alternativas com prós/contras quando relevante
- Sugerir testes unitários junto com o código de produção
- Focar em soluções **incrementais e entregáveis** (vibe coding)

## Modo Vibe Coding

Quando o usuário estiver em modo vibe coding:
1. Gere o código **completo e funcional** sem esperar confirmação intermediária
2. Priorize exemplos reais sobre explicações teóricas
3. Entregue arquivos prontos para copy/paste ou scaffold
4. Sugira o próximo passo ao final de cada entrega
