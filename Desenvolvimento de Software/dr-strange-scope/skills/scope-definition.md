# Skill: Definição de Escopo

## Template de Escopo

```markdown
## Escopo do Projeto: [Nome]

**Objetivo:** [Uma frase clara sobre o que o projeto entrega]
**Problema resolvido:** [Dor específica do usuário]
**Público impactado:** [Personas afetadas]

---

### ✅ Escopo Incluído (MVP)
- [funcionalidade 1]
- [funcionalidade 2]
- [funcionalidade 3]

### ❌ Escopo Excluído
- [o que não entra no MVP e por quê]

### 🔜 Próximas Releases
- [funcionalidades futuras identificadas]

---

### MVP Sugerido
[Descrição do menor conjunto funcional que gera valor real ao usuário]
**Critério de lançamento:** [o que precisa estar 100% para o MVP ir ao ar]
```

## Escolha da Metodologia

| Contexto | Metodologia Recomendada |
|----------|------------------------|
| Time pequeno, escopo incerto | Kanban |
| Time estruturado, entregas por sprint | Scrum |
| Produto em descoberta contínua | Lean + Kanban |
| Time distribuído, alta frequência de release | XP / CI-CD focado |
| Necessidade de previsibilidade com flexibilidade | Scrum + Kanban híbrido |

Sempre justificar a escolha com base no contexto levantado no discovery.

## Validação do Escopo

Antes de avançar para o backlog, confirmar com o usuário:

```
O escopo acima está correto?
- Algo deve ser adicionado ou removido do MVP?
- As exclusões fazem sentido para o contexto?
- O critério de lançamento é realista?
```
